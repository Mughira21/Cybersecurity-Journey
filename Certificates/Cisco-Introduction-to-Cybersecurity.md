# Cisco – Introduction to Cybersecurity

## Overview
This document summarizes my learning from the **Introduction to Cybersecurity** course by Cisco Networking Academy. It covers basic cybersecurity concepts, types of attacks, network security, and protection techniques.

---

## 🦠 Types of Malware

- **Spyware**: Tracks user activity, including keystrokes.
- **Adware**: Displays unwanted pop-up advertisements.
- **Backdoor**: Bypasses authentication to gain unauthorized access.
- **Ransomware**: Encrypts data and demands payment to restore access.
- **Scareware**: Fake warnings to trick users into installing malicious software.
- **Rootkits**: Provide hidden, persistent access to a system.
- **Virus**: Attaches to files and spreads when executed.
- **Trojan Horse**: Appears legitimate but contains malicious code.
- **Worms**: Self-replicating malware that spreads across networks without user action.

---

## 🌐 Online Identity
Even without social media, users have an online identity (name, date of birth, etc.).  
Using strong and unique usernames and credentials is important for security.

---

## 🔐 McCumber Cube (Security Model)

- **Confidentiality** → Protect data (encryption, authentication)
- **Integrity** → Prevent unauthorized changes
- **Availability** → Ensure data is accessible when needed

---

## ⚠️ Malware Symptoms

- Slow system performance  
- High CPU usage  
- Frequent freezing  
- Unknown files appearing  
- Network issues  
- Data being accessed without permission  

---

## 🎭 Social Engineering Attacks

- Manipulating people to gain access  
- Examples:
  - Pretending to be someone else  
  - Urgent requests for passwords  
  - Tailgating (physical access)

---

## 🌊 Network Attacks

### 🔹 DDoS (Distributed Denial of Service)
- Overwhelms systems with massive traffic using infected machines (botnets)

### 🔹 Man-in-the-Middle (MITM)
- Intercepts communication between two users

### 🔹 SEO Poisoning
- Malicious websites ranked higher in search results to trick users

### 🔹 Password Attacks
- Brute force (trying multiple combinations)  
- Dictionary attacks  
- Rainbow table attacks (cracking hashed passwords)

---

## 🧩 Security Vulnerabilities

### Hardware
- Example: Rowhammer attack (memory manipulation)

### Software
- Bugs or unpatched systems can be exploited  
- Solution: Regular updates and patches

---

## 🪙 Cryptojacking
- Using someone’s system secretly to mine cryptocurrency

---

## 🛡️ Protecting Devices & Networks

- Enable firewall  
- Install antivirus/antispyware  
- Update software regularly  
- Use strong passwords  
- Configure browser security  

---

## 📡 IoT Security

- IoT devices are vulnerable due to weak security  
- Tools like **Shodan** can identify exposed devices  

---

## 📶 Wi-Fi Security

- Use **WPA2/WPA3** (avoid WEP)  
- Change default router credentials  
- Disable unused features  
- Use MAC filtering  

---

## 🌍 Public Wi-Fi Risks

- Use **VPN**  
- Avoid sensitive activities  
- Use HTTPS websites  
- Disable auto-connect  

---

## 🔵 Bluetooth Security

- Turn off when not in use  
- Use hidden mode  
- Remove paired devices  

---

## 🔑 Password Security

- Use passphrases  
  - Example: `AcatThatLoves3Dogs!`  
- Avoid simple or common passwords  

---

## 🔒 Data Protection

- Encrypt sensitive files  
- Backup data (cloud/offline)  
- Permanently delete data using secure tools  

---

## 📜 Terms & Privacy Awareness

- Understand:
  - How data is collected  
  - Who can access it  
  - What happens when you delete your account  

---

## 🔎 Port Scanning

- Used to find open ports and services  
- Tools: **Zenmap (Nmap)**  
- Helps identify vulnerabilities  

---

## 🛡️ Security Tools

### IDS (Intrusion Detection System)
- Detects suspicious activity (alerts only)

### IPS (Intrusion Prevention System)
- Detects and blocks threats automatically  

### Firewall
- Blocks unwanted traffic  

---

## 📊 NetFlow

- Monitors network traffic patterns  
- Tracks:
  - Source & destination IP  
  - Ports used  
  - Data volume  

---

## 🧪 Penetration Testing Process

1. Planning (gather information)  
2. Scanning (find vulnerabilities)  
3. Gaining access (exploits, attacks)  
4. Maintaining access  
5. Reporting findings  

---

## 🧰 Security Systems

### SIEM
- Collects and analyzes logs in real-time  

### DLP (Data Loss Prevention)
- Protects sensitive data:
  - Data in use  
  - Data in motion  
  - Data at rest  

---

## 🧠 My Key Takeaways

- Cybersecurity is about protecting systems, networks, and data  
- Human error (social engineering) is a major risk  
- Networking knowledge is critical for security roles  
- Tools and monitoring systems play a big role in defense
